其他利用途径（FILE_plus and vtable）

前面介绍了利用FILE实现任意地址读写，这里介绍利用FILE_plus的漏洞实现控制流劫持。

1. FILE_plus介绍

FILE_plus与FILE的唯一区别是增加了一个指针，指向虚函数表。

2. 虚函数表（Virtual function table）

一个存储了若干函数指针的表，常见于C++，主要用于实现运行时多态（在运行时根据对象的实际类型，决定调用哪个函数）。

每个类有自己的虚函数表，通常位于程序的数据段（如.data、.rodata）。

对象调用函数时，通过虚函数表指针+偏移量，找到对应的函数指针，再调用该函数。偏移量在编译时确定，运行时再根据对象所属的不同类，到对应的类函数表中查找。

FILE_plus的虚函数表指针，也指向了一个存放函数指针的表。

而代码中会有类似call QWORD PTR [r15+0x38]的命令，r15是取出的指针值，0x38是偏移量，以此调用函数。

3. 伪造vtable

对于FILE_plus，会取fp.vtable作为r15的值。如果可以自己伪造vtable，然后修改fp.vtable指向伪造的vtable，就可以劫持程序流。

困难1：锁指针_IO_lock_t *_lock

如果想覆盖*vtable，需要将FILE_plus完全覆盖，其中锁指针需要指向一个可写且值为0的地方，才不会段错误或阻塞。

困难2：即使成功将*vtable指向了伪造的vtable，并解决了锁的问题，也无法保证跳转到目标函数。因为libc会检测vtable是否位于专门为vtable预留的区域中，伪造的会被识破。

我们可以做的：

  • 在vtable区域内寻找合适的函数。
  • 尝试调用IO_wfile_overflow，进而调用do_allocbuf。
  • do_allocbuf中会使用另一个指针：FILE->_wide_data。
