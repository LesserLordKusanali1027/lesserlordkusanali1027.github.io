正则表达式学习

发表于2026-02-13|更新于2026-02-21|战编译
|浏览量:

最近试图战编译，词法分析要用到正则表达式，所以浅浅学一下，配合在线工具，参考别人的笔记。这个网站也挺有趣，可以玩一玩。

正则表达式是什么

正则表达式 (regex)，是一组由字母和符号组成的特殊文本。

可以描述某类特定格式的字符串，可用于从文本中找出满足的该格式的字符串，也可用于词法分析匹配token。

比如：[a-zA-Z]{3,5} 匹配长度 3∼53\sim 53∼5 的字母串
输入 Only My Railgun
匹配输出 Only, Railg

简单匹配

就是一对一匹配，写啥匹配啥，Only、My、Railgun这种。

元字符匹配

元字符是正则表达式的基础。

元字符 含义
. 匹配单个字符
[] 匹配单个括号内任意字符
[^ ] 匹配单个除了括号内的字符
* 匹配 ≥0\ge 0≥0 个重复的 * 前的字符
+ 匹配 ≥1\ge 1≥1 个重复的 + 前的字符
? 匹配 0/1 个 ? 前的字符
{n,m} 匹配 n≤num≤mn\le num \le mn≤num≤m 个大括号前的字符
(xyz) 匹配与xyz完全相等的字符串
| 或运算
\ 转义字符，用于保留字符`[ ] ( ) { } . * + ? ^ $ \
^ 定位符，要求位于字符串开头(或行开头，取决于模式)
$ 定位符，要求位于字符串结尾(或行开头，取决于模式)

点字符 .

匹配任意单个字符，是否包括换行符取决于格式

  • .ly
输入 Only My Railgun\nly
匹配输出，.包括换行式 nly, \nly
匹配输出，.不包括换行式 nly

字符集 []

  • [lM]y，匹配后面是y前面是l或M的字符串
输入 Only My Railgun
匹配输出 ly, My

否定字符集 ^

  • [^M]y，匹配后面跟着y但前面是除了M的其他字符的字符串
输入 Only My Railgun
匹配输出 ly

* 号

匹配字符集中 0 个或多个字符组成的串，会尽可能多地匹配

  • a*b，一处匹配，a*把a全拿了
输入 aaaaab
匹配输出 aaaaab
  • a*ab，(其实与a+b等价)，一处匹配，后面ab占了一个a，所以a*拿了四个a，也是能力范围内全拿
输入 aaaaab
匹配输出 aaaaab

? 号

  • [i]?l，0 或 1 个 i 加 l
输入 Only My Railgun
匹配输出 l, il

区间量词 {}

  • [0-9]{2,3}，匹配 2 到 3 位的数字串
输入 1 23 456 7890
匹配输出 23, 456, 789
  • [0-9]{2,}，匹配至少 2 位的数字串
输入 1 23 456 7890
匹配输出 23, 456, 7890
  • [0-9]{4}，固定匹配 4 位数字串
输入 1 23 456 7890
匹配输出 7890

子表达式 ()

()内的内容会被看成一个整体。

  • (10)+，会匹配 ≥1\ge 1≥1 个 10 拼成的字符串
输入 10 1010 1111
匹配输出 10, 1010

或运算 |

  • (l|M|n)+y，匹配后面是y，前面是l/M/n三个字符重复 ≥1\ge 1≥1 次的字符串
输入 Only My Railgun
匹配输出 nly, My

转义字符 \

如果要匹配 {、}、[ 这些被用作元字符的工具字符，需要加上转义符 \

  • \(abc\)\*
输入 (abc)* abc
匹配输出 (abc)*

定位符 ^ 和 $

^指定匹配位于开头的字符串，$指定位于结尾的字符串。至于是整个字符流的开头/结尾还是行的，都有可能。

  • ^(a|b|c)+，只匹配位于开头的
输入 aa ab cc
匹配输出 aa
  • (a|b|c)+$，只匹配位于结尾的
输入 aa ab cc
匹配输出 cc

简写字符集

简写 含义
\w 匹配所有字母数字下划线，等价于 [a-zA-Z0-9_]
\W 匹配所有非字母数字下划线，等价于[^\w]
\d 匹配数字，等价于[0-9]
\D 匹配非数字，等价于[^\d]
\t 制表符，ASSCII码为0x09，是Tab输入的符号
\n 换行符，ASSCII码为0x0a
\v 垂直制表符，ASSCII码为0x0b
\f 换页符，ASSCII码为0x0c
\r 回车符，ASSCII码为0x0d
" " 空格，ASSCII码为0x20
\s 所有空白字符，包括[ \t\n\v\f\r]，可能还有其他空白字符
\S 所有非空白字符，等价于[^\s]
\p 匹配\r\n，在 Windows/DOS 中用来表示文本行结束

贪婪匹配与惰性匹配

正则表达式默认采用贪婪匹配，会匹配尽可能长的子串，Flex 总是采用贪婪匹配进行词法分析。

相对的是惰性匹配，可以加?来实现。注意惰性匹配并不是匹配尽可能短的字符串，而是从当前位置开始，尽可能少地消耗字符，同时保证整个正则表达式能够匹配成功。

  • [\w]*gun，贪婪匹配
输入 Only My RailgunRailgun
匹配输出 RailgunRailgun
  • [\w]*?gun，惰性匹配，从左往右匹配时遇到第一个gun就结束了
输入 Only My RailgunRailgun
匹配输出 Railgun, Railgun
