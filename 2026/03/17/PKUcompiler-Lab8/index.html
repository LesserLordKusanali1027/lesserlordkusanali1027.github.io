Lab8，飞跃

自卑、胆小、没用的我，能与才智出众之人交锋的手段唯有一个，那就是花更多的时间

啊啊啊我要变强 (ノTДT)ノ

概述

目标是实现函数、全局变量。

  • 函数定义 (函数声明暂时不需要)
  • SysY 库函数要能直接使用
  • 全局变量

语义规范：

  1. 整个 SysY 程序文件对应语法规则中的 CompUnit，全局变量/常量和函数的定义都在它下面，这里面：
    • 必须有且只能有一个 int main() 的函数定义
    • 顶层变量/常量声明不能重名 (这和局部的规则一样)；函数也不可以重名，也就是说不支持重载函数。
    • 顶层变量/常量、函数的作用域为整个文件
  2. 全局变量：
    • 局部变量可以覆盖全局变量
    • 局部变量名可以和函数名相同
    • 全局变量声明中指定的初值表达式必须是常量表达式
    • 未显式初始化的全局变量，初始化为 0
  3. 函数
    • 定义的函数名不能和 SysY 库函数名相同
    • 现在有两种返回类型了，int 和 void ，如果…算了，到时候再看吧

把这个结构塞进原有的多层符号表有些困难，不论是数据结构还是方法都很不一样，所以另开一个全局函数符号表类，并实现以下方法：

IR 生成

增加以下内容：

  1. 增加了 call 指令，有两种格式，有返回值的和无返回值的，用一种指令类型统一表示

    • call 函数名(参数,...)
    • %num = call 函数名(参数,...)

  2. void 的返回指令 ret ，只有一个操作码，再新增一个类型吧

  3. 为 FunctionIR 增加成员 std::vector<std::string> parameters 来记录参数

  4. 在 Lab4 中我增加了 lval_mode 用来告诉 LValAST 节点应该 store 还是 load，但该状态是在 Exp 的上一层节点设定的，这个做法有个重要前提是这些上层节点不会嵌套调用，否则 lval_mode 多个状态会覆盖、丢失

    没有函数时是可以的，但是函数的嵌套调用带来了状态覆盖的问题，比如 x = add(add(1)); 多个函数会为 lval_mode 重复赋值，所以增加栈 lval_stk 记录过去的状态以便恢复

    PS：其实也有不用栈的方法，但我觉得用栈可扩展性更强

  5. call 指令需要由两个节点共同确定，一个确定参数，另一个确定其他。而确定参数的节点又会遇到 add(sub(1,2),mul(2,3)) 这种嵌套调用，也需要用栈记录前面的 call 指令，以便处理完嵌套的函数的 call 后还能把外层函数的 call 补完。

此外，操作上需要注意：

  1. 函数参数的处理：这里为参数又 alloc 了新的内存空间，参数命名为 %...，内存空间为 @...，与其他内存空间的前缀保持一致 (@)。虽然直接用 %x 而不是 %0 也可以，但这么做是为了方便 RISC-V 生成。
  1. 对于 void 还没写 return; 的函数，需要我们自己在 IR 加上 ret，我采取的方法是判断最后一个基本块 (尚未加入函数) 的最后一条指令是不是 ret，如果不是就加上 ret。

RISC-V 生成

手册写得很详细，照着做就 ok 了，主要做的事情有：

  • 实现了 stack_setup 函数，为每个函数计算存返回地址的栈空间、存局部变量的栈空间和存参数的栈空间，这三者加起来再对齐到 16 字节就是该函数的栈帧。
  • 实现了 call 时的参数传递：前 8 个参数被放到 a0~a7 中，后面的参数放到调用函数的栈帧中。
  • 实现了被调用函数的参数获取：前 8 个参数把 a0~a7 sw；后面的参数先从调用函数的栈帧里 lw，再 sw 到局部变量中。

语义分析

SysY 的库函数是可以不加声明直接使用的，所以在语义分析中，需要先在函数符号表中添加库函数信息，使得遇到调用库函数的地方后可以正确识别，遇到与库函数重名、库函数参数不符的情况也可以准确报错。

由于库函数并不是我们自己实现的，这里另建一个函数声明表 FunctionDeclTable ，用来放之后自己的函数声明。

目前先手动在语义分析开始向 FunctionTable 中加入库函数名、返回值类型和参数个数信息吧。封装在 def_lib_func 中。注意这里的 int[] 类型要在 Lab9 才实现，所以先不用管。

库函数信息如下：

  1. 读入整数 int getint()
  2. 读入字符 int getch()
  3. 向数组读入一串整数 int getarray(int[])
  4. 输出整数的值 void putint(int)
  5. 输出整数的值对应的 ASCII 字符 void putch(int)
  6. 输出数组中一定数量的整数 void putarray(int, int[])
  7. 计时函数，性能测试中有用到

语义分析

全局常量、变量的处理方法与局部的基本一样，唯一的不同点是：全局变量如果有赋值语句，那么该语句一定是常量表达式且要能算出结果，这与局部常量的赋值方式相同了。

后续的 IR 生成中要直接为有赋值语句的全局变量赋值，所以需要存下表达式的结果，有两种选择：

  • 另建一个数据结构，且能传递给 IR 生成器
  • 修改 AST 树

选择后者，这属于另一种常量折叠了，但比之前的常量折叠改的节点要多得多。

总结

关键内容：

  • 实现了函数的定义和调用
    • 为语义分析添加了全局函数符号表 func_def_table ，用来记录函数信息并检查错误
    • IR 加入了 call 指令
    • RISC-V 设计了新的栈帧，并利用栈来保存返回地址、传参
    • 函数的嵌套调用，为我之前设定的部分状态变量带来了麻烦，不过我暂时用 std::stack 很好地解决了它们
  • 实现了 SysY 库函数的调用
    • 在语义分析中用 func_def_table 来记录库函数的信息，手动放入。同时增加了 func_decl_table 为以后会出现的函数声明语句做准备。
    • IR 加入了 FunctionDeclIR 来记录函数声明，将库函数声明手动放入
  • 实现了全局常量、变量
    • 全局常量依旧使用常量折叠，在语义分析全部处理掉
    • 为全局变量新增了 GlobalIR，之后用 RISC-V 的 data 部分存放，同时引入了地址加载指令 la 来读写 data 中的数据

函数的加入将程序拔高了一个维度，全局变量和 SysY 库函数的加入也大大增加了程序能做的事情，这真的是一次飞跃了。
