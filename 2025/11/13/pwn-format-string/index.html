Format String Exploits (格式化字符串利用)

发表于2025-11-13|Pwn.college
|浏览量:

格式化字符串漏洞利用

Introduction

printf传参方式：

printf接收一系列参数，第一个参数被当成格式化字符串，后面的参数用于拼入格式化字符串。

但printf盲目信任第一个格式化字符串，如果我们可以控制格式化字符串，事情就失控了。

printf的传参约定：RSI,RDX,RCX,R8,R9,栈。示例如下，在gdb中查看。

在call printf@plt处下断点，查看寄存器内容，1~5分别位于rsi,rdx,rcx,r8,r9中，与普通函数使用寄存器传参的规则相同。

rdi指向格式化字符串的地址：

而剩下的3个参数，会通过栈传递，6、7和字符串地址被放在了栈顶。注意： 虽然6、7都是 4 bytes 的int，但放在栈上都占用 8bytes 空间。

下面是格式化字符串漏洞存在的唯一条件：用户可以控制一个格式化字符串。即使没传输出参数，printf也会从寄存器和栈上取。

Leak Data

如上面所见，printf取参数会以8字节为单位，寄存器和栈都如此，然后根据%...进行不同的解析。由此，可控制泄露信息的格式。

一些格式化字符的例子：

  • %c 一个char字符，1byte
  • %d、%i int显示为十进制数，4byte
  • %x 显示十六进制数，4bytes
  • %hx 显示十六进制数，2bytes (一些控制位数的前缀)
  • %hhx 显示十六进制数，1bytes
  • %lx 显示十六进制数，8bytes
  • %s 读取8bytes指针值，输出指向的字符串
  • %n$.. 指定输出第n个参数

程序：

使用不同的格式化字符，对第一个传入参数0xBEEF1337做格式上的变换。

使用%p可以将8字节按十六进制输出，前5个是寄存器内容，后面是栈上内容。

使用%n$p指定输出第n个参数，再用%s将其作为地址解析，就能泄露secret_value了。

Writing Data

基本方法：

%n会将当前输出的字节数写入某个指针指向的地址，可用来修改内存。

先确定print_flag在栈上的位置是printf的第几个参数。具体分析如下，输入a%41$n即可成功将print_flag改为1。

%n的一个特点：

%n只会修改指针指向地址处的4字节，而不是修改8字节。如下，高4字节的12345678被保留，低4字节被改为00000001，就是说%n默认面对int指针。


其他技巧：

  • 如果用户可控制的格式化字符串位于参数所在栈帧，可以将自己输入的内容作为指针，然后用%n解析，可以指向很多地方。
  • 帧指针rbp之间的关系也可以利用，比如下面利用rbp1指向rbp2存储处，来修改存储的帧指针值。
  • 更灵活的写入，%n只能写4字节，不够灵活，添加前缀。
    • %hhn 写1字节
    • %hn 写2字节
    • %ln 写8字节

控制%n写入的数值：

可以使用空白填充语法，下面会在输出%x前先填充若干空格，使最终总字节数达到1145258561，并写入buf。

但这太慢了，4字节可以达到很大的数字，以1字节为单位来写是更好的选择。

Copy Memory

dynamic padding（动态填充）：

前面使用%65x这种方式进行填充，保证输出特定字节数；动态也是这个作用，但不使用具体数字（如65），而是用%*来从某个参数获取填充长度。

比如：

  • %*10$c中，*表示填充值从某个参数获取，10$表示取第10个参数（可能转换为int）作为宽度值，c就是输出。
  • %11$n执行写操作。

这个式子会将第10个参数的内容写到第11个参数指向的地址处。

但若要写的内容太大，容易爆掉。

遗留问题：%c从哪取不知道。
